## Шаг 1 — Данные (25%)

### Подход

Базовый датасет: **[tatsu-lab/alpaca](https://huggingface.co/datasets/tatsu-lab/alpaca)** — открытый датасет с 52K примеров instruction-response

Поля датасета: `instruction`, `input` (дополнительный контекст, часто пустой), `output`. При наличии `input` он конкатенируется к `instruction` через перенос строки.

Целевой стиль ответов: лаконичные, структурированные ответы с bullet-points, без вводных фраз. Стиль технического наставника: прямо по делу, с формулами и примерами где уместно.

### Этапы подготовки

**Загрузка и фильтрация качества:**
- Минимальная длина инструкции: 20 символов, максимум: 400
- Минимальная длина ответа: 30 символов, максимум: 800
- Удаление примеров со ссылками (добавляют шум)
- Фильтрация примеров со спецсимволами в инструкции

**Стилизация ответов:**
- Удаление вводных фраз ("Sure, ", "Of course, ", "Certainly, " и т.д.)
- Преобразование длинных ответов в структурированный формат с bullet-points
- Применение случайного из двух шаблонов стилизации

**Синтетические примеры (50 штук):**
- 30 пар по ML/DL тематике: градиентный спуск, LoRA, attention, метрики и т.д.
- 20 пар по общим CS темам: рекурсия, TCP/UDP, Docker, REST API
- Написаны в целевом стиле с нуля

**Дедупликация:** MD5-хеш инструкций, удаление точных дубликатов

**Финальный датасет:**
- 220 примеров (200 train + 20 val)
- Формат JSONL: `{"instruction": "...", "response": "..."}`
- Средняя длина инструкции: ~80 символов
- Средняя длина ответа: ~250 символов

## Шаг 2 — Fine-tuning (50%)

### Архитектура решения

**Базовая модель:** `mistralai/Mistral-7B-Instruct-v0.2`  
7B параметров

**Квантизация (4-bit NF4):**
- BitsAndBytesConfig с `bnb_4bit_quant_type="nf4"`
- Double quantization для дополнительной экономии (~0.4 bit)
- Вычисления в float16
- VRAM после загрузки: ~6 GB

**LoRA конфигурация:**
```
rank r = 16          # Баланс качество/параметры
alpha = 32           # Масштаб = alpha/r = 2
dropout = 0.05
target_modules: q_proj, k_proj, v_proj, o_proj,
                gate_proj, up_proj, down_proj
```

**Гиперпараметры:**
```
learning_rate = 2e-4
epochs = 3
batch_size = 2, grad_accumulation = 4  → effective batch = 8
warmup_ratio = 0.05   (5% шагов)
scheduler = cosine
optimizer = paged_adamw_8bit
max_seq_length = 512
fp16 = True
```

## Шаг 3 — Оценка (25%)

### Тестовая выборка

15 примеров, **не входящих в train** — по тематике ML/DL/NLP. Для каждого написан эталонный ответ в целевом стиле.

### Метрики

**ROUGE (Recall-Oriented Understudy for Gisting Evaluation):**
- ROUGE-1: перекрытие отдельных слов
- ROUGE-2: перекрытие биграмм
- ROUGE-L: наибольшая общая подпоследовательность
- Быстро вычисляется, хорошо отражает лексическое совпадение

**BERTScore:**
- Косинусное сходство эмбеддингов токенов (через BERT)
- Улавливает семантическую близость, а не только точное совпадение
- Более устойчив к перефразировкам
- Precision, Recall, F1

### Как интерпретировать результаты

ROUGE-L < 0.2 на свободной генерации — норма. Дообученная модель может генерировать качественные ответы с другими формулировками, что даёт низкий ROUGE. BERTScore устойчивее в таких случаях.

Основные маркеры успешного дообучения:
- Ответы стали короче и структурированнее (меньше воды)
- Появились bullet-points и формульные обозначения
- Исчезли вводные фразы ("Sure, ", "Of course, ")
- BERTScore F1 вырос (семантическая близость к эталону)

## Выводы и рассуждения

### Что работает хорошо

**200 примеров достаточно для стилевого сдвига.** Стилевое дообучение (instruction tuning поверх уже обученной модели) требует меньше данных, чем обучение с нуля. Модель уже умеет отвечать — мы только корректируем формат.

**QLoRA на T4 — практичное решение.** 7B модель в 4-bit + LoRA адаптеры размером 80 MB — реалистичный workflow для одиночного GPU. Без квантизации T4 бы не справился.

**Синтетические примеры помогают.** Вручную написанные пары в точном целевом формате задают "якорь" стиля лучше, чем стилизованные примеры из долли.

### Ограничения

**ROUGE/BERTScore — неполная оценка стиля.** Метрики меряют близость к эталону, но не структурность и лаконичность. Дообученная модель может давать лучшие ответы с более низким ROUGE (другие формулировки при том же смысле). Полноценная оценка требует human evaluation или LLM-as-judge (GPT-4 оценивает по критериям).

**Catastrophic forgetting при малом датасете.** 200 примеров на 3 эпохи — риск деградации на задачах вне целевой тематики. Решение: больший датасет + меньше эпох.

**15 тестовых примеров — нет статистической значимости.** Разброс на таком размере выборки большой. Для надёжных выводов нужно 100+ примеров.
